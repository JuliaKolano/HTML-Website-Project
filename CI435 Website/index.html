Week 7: Flexible Images

Published on November 18, 2021

After today's tutorial I have managed to create a separate webpage, in order to practice different techniques when including images on a website. And so, as the first example I created a full-sized clipped photo. I achieved this by using the overflow attribute and setting it to hidden, this means that as the viewport is resized, the image will get cropped and a part of it will dissapear from the viewport.

Secondly, I have also created a flexible photo. I did this with the width property and setting it to 100%, which means that the image will change its size when the viewport is resized, so that the whole image is displayed.

To take the second example further, I have also added text to the image as a third example. I've set the width attribute to 33%, so that when the viewport is resized, the image is always displayed on one thirds of the screen, and I've also set the float property to the left, so that the image stays on the left and the text always wraps around it on the right side.

Finally, as the last example, I've also made a flexible video conatiner. This was done by setting the width and height attributes to 100%, so that the whole video container is displayed as the viewport is resized.

Additionally, as a further reading for this week I have read the website called Fluid Images by Ethan Marcotte. After reading the article I have found out that images can be restricted in their size to never be bigger than the container they were originally in, this is done by using the max-width: 100% rule in CSS. However, some older browsers don't support this rule, and therefore we have to use other methods to create fluid images in these kinds of browsers, including using the width: 100% rule or using JavaScript. As well as this, I read a Wikipedia Webpage about Copyright. And so according to Wikipedia, copyright is a type of intellectual property, which means that only the author of a certain creative property has the full access to that original property. So I need to be careful when chosing images for my website, and I need to make sure to find copyright-free images if I want to use them on the internet. Finally, I has a look at the Creative Commons Website as well. There, I've learned that creative commons licenses provide authors of creative property to be able to publish their work and provide certain rules for when other people want to access the property for their own use. So, the best idea for me is to find images under the creative commons licenses that let me use the pictures as long as the original author is referenced.

Therefore, this week I have learned the following skills:

  1. Creating a clipped full-size image
  2. Creating a flexible image
  3. Creating a flexible image with text wrapping
  4. Creating a flexible video container

Week 6: Making a Responsive Webpage

Published on November 11, 2021

After today's tutorial I have found out that it's important to sketch a layout of a webpage first, so that it can be easier to implement the design for the page later on. Therefore, after I had an idea of how I wanted to lay out my page, I added the viewport meta tag to make that my website displays on a phone in the same way I have designed it on a desktop. As well as that, I also added the class of banner to the head tag, so that I can style its navigation in CSS.

In order to make my website more responsive I used the grid-area property for my webpage depending on the screen widths. I did that so I can style the layout for 3 different scenarios: for small screens such as mobile phones, for medium-sized screens such as tablets, and for wide screens such as desktop computers. I have also decided that it would be better if I moved the 'go to top' anchor to the footer tag instead, because I thought that it would allign better with the grid layout I wanted to implement. For the same reason, I also changed the paragraph allignment to the left, instead of right, so it would allign better with the grid.

For the design of my website, I have decided to use the mobile first approach, so I started with styling everything vertically, using the display attribute and setting it to block. For small-screen devices I also used the display attribute, so that I could hide the timetable form the webpage because it wouldn't properly fit and allign on it. After I was done with the verical design of the webpage, I went onto designing it for the wide-screen devices and so I added a horizontal menu at the top of the page for the contents, using the display attribute and setting it to inline. And finally, I made the images flexible, so that they would size properly depending on the width of the page that it's viewed on.

Furthermore, as part of this week's reading I looked at the grid by example website, which provided me with a lot of useful layouts that I could take inspiration from for my own webpage. I have also read the responsive web design article by Ethan Marcotte. By reading through the article I have found out that web developers used to make different versions of websites for different devices, so that the pages would display properly and be easy to read. Howeve, nowadays we shouldn't do that, and instead we should make our websites more responsive and use media querries to change the layou of the webpages depending on the size of the screen. Additionally, after reading through the Developer Mozzila Webpage about grids I have found out that the grid layouts for webpages are very useful for separating content into specific blocks around the page. It helps with the website being more responsive because the grids move around based on the available space. Finally, I also found the information on the Developer Mozzila Webpage about media querries useful because it explained that media querries help to apply different CSS elements to the webpage based on device characteristics and parameters, which helps with responsive web design.

Therefore, this week I have learned the following skills:

  1. Sketching wireframes of different width screen layouts
  2. Adding the viewport meta element to the head of the document
  3. Adjusting HTML if necessary
  4. Organising the stylesheet
  5. Setting up a responsive grid
  6. Styling a small-screen layout
  7. Styling for different screen widths using media querries
  8. Hiding elements
  9. Making images flexible
  10. Testing responsive webpages

Week 4: CSS Basics

Published on October 28, 2021

After today's lab, I managed to use the normalize.css stylesheet to help my browser render the elements on my webpage within the standards used on most of the sites. As well as using this stylesheet, I have also used and linked my own stylesheet to all of my webpages, which I have started to write in during this workshop. This was done so that I would be able to personalise the look of my website using CSS, rather than leave it with default settings for display.

As part of using CSS to style my webpages, I have decided to change the h1 heading to a red colour so that it stands out from the rest of the text. I've done this as a mean of experimentation on my website, and I might change it during further workshops, so that my webpage is more pleasing to look at. I have also decided that I would convert the links, so that when a user of the website hovers over them, they would change colour, so that it's easier for them to tell which parts of the page are links, and which are plain text. Furthermore, as a cosmetic choice I have also decided to change the colour of the rest of the links, so that they would be easier on the eyes of the reader.

Since I used a table in my html from the last workshop, without the use of a stylesheet, I had to include CSS in my html, which produced an error when I tried to validate my code. However, now I have linked a stylesheet to my webpages, so I have moved that part of CSS into the stylesheet, instead of the html and managed to get rid of the error. Also, to improve the navigation of my webpage and make the weekly journal posts stand out, I have changed the colours of the headers of the articles to grey, using the class attribute, so that only the h3 headers of the journal posts changed their styling, and not all of the other h3 headers.

Finally, I have also customised the whole body section of the document, in such a way that the font was changed and the whole paged got centred towards the middle. I have done this so that my page is easier to read for the user of the website.

As part of my further reading for this week, I have read through the CSS reference on the MDN Web Docs website, which included the information about the rules of CSS and the proper syntax that should be used in the stylesheets. Furthermore, by readding this webpage, I have found out that there are different types of selectors. I have decided to use mainly the class and id selectors; the id selector can only be used once per page, however the class selector can be used multiple times, and it will work anytime it's called within the page. The website also mentions how multiple selectors can be grouped by using a comma between them. I have also found this website to be a good source of different CSS functions because it includes an index, so I can always come back to this website if I want a reminder of all of the functions available in CSS.

Therefore, this week I have learned the following skills:

  1. Using stylesheets and liking them to my webpages
  2. Styling different elements of my webpage using CSS
  3. Anchoring pseudo-classes

Week 3: More HTML Markup

Published on October 21, 2021

After today's lab, I have found out that it's possible to use nested lists on a webpage, and that it can add more structure to a page. However, I decided that I want to keep my layout more simple, not add nested lists because I believe my website doesn't require them at the moment. But I have gained useful knowledge about them, so I would be able to implement them in the future if I wanted to.

Furthermore, I have also created a table after the contents section on my website. I did this to include my time table as an additional part of my webpage. It doesn't directly link to the main part of the webpage, and so I wrapped it in an aside tag to make that clear.

Another useful thing that I have leaned today is the use of quotes on a webpage. Quotes are useful because it's possible to include some extra information on a page from external source, which are not written by the author of the page. The way I used the blockquote tag was to quote a definition from an external website about screen readers, so that whoever is reading the website can have a clear idea of what exactly screen readers are.

The last thing I have learned during the tutorial is how to link an image from an external source onto my wepsite. This is done using the img tag inside of the a tag. Using this knowledge I have added a link to the HTML logo close to the top of my website.

I have also done some further reading on the Mozilla webpage about document and website structure. This website has helped me to correct some errors I have made on my wesite originally. For example I wanted to put the aside tag for the table I've made outside of the main tag, which after reading through the website I've found out it's bad practice. Therefore, I have corrected that, and now my websites navigation is much more clearer. Reading further into this website has also made me aware of the br tag. Before I knew about this tag, I have used the p tag to start a new line in the caption section below the pictures on my webpage. However, I now changed that to a br tag to create a new line, which is much better practice in this instance.

Another website I have used for further reading is the Mozilla webpage about creating hyperlinks. From this website I have found out about the title attribute, which helps to add extra information about a hyperlink, so that the reader of the website can have a better idea of what is included in that hyperlink, and why they should or shouldn't click on it. This website also talked about the difference between the absolute and relative links, which helped me to understand when each of them should be used and how to properly use them. Also it mentioned about some errors that could occur if the location of a file changes when using the relative links, making me understand what to look out for when using this type of links.

The last reading I've done for this week was to gain more knowledge about screen readers. Therefore, I have found a definition on the Ability Net webpage that states:

"A screen reader is a technology that helps people who have difficulties seeing to access and interact with digital content, like websites or applications via audio or touch."

AbilityNet

The information on this website was very useful to me because it helped me understand how people with visual impairments access the internet, and so it helps me to develop my website in a way that would make it more accessible to different kinds of users.

Therefore, this week I have learned the following skills:

  1. Creating nested and definition lists
  2. Adding a table to my learning journal
  3. Quoting text using the blockquote tag
  4. Making an image into an external link
